  • Patterns
  • Tables (테이블)
  • 데이터 입력형

데이터 입력형

• 데이터 입력형 테이블은 사용자가 데이터를 입력하거나 수정할 수 있도록 구성된 테이블이다. 입력된 데이터가 유효한지 확인하기 위해 유효성 검사를 하도록 설정할 수 있다.

구조

테이블 입력형

• 인풋, 셀렉필드와 같이 입력 할 수 있는 폼들로 구성되어진 테이블이다.

• 테이블 셀(td)안에 들어갈 수 있도록 입력 텍스트 수를 지정해야 한다. (두 줄 X)

• 테이블 셀(td) 안에 들어가는 텍스트를 고려하여 사이즈를 고려하여 텍스트가 잘리지 않도록 해야 한다.

• 스크롤 테이블 형태에서는 입력폼을 사용할 수 없다.

a. 세로형_데이터 입력형
b. 가로형_데이터 입력형

사용법

  • • 세로형 테이블에서 유효성 검사시, 에러가 있을 때 토스트 알럿으로 오류를 표시한다.

  • 에러가 발생한 셀에 에러 표시(빨간색 테두리)를 하여 오류가 발생한 셀을 강조합니다.

  • • 가로형 테이블에서 유효성 검사시, 에러가 발생한 셀 하단에 오류메시지를 표시한다.
  • • 테이블 섹션 타이틀의 경우 테이블 버튼은 최대 4개까지만 노출한다.

  • 버튼 개수는 최소한 적게 사용하는 것을 원칙으로 한다.

  • • 테이블 페이지네이션 조건은 데이터 개수가 20개를 초과하는 경우에만 노출한다.

  • 페이지네이션에 대한 자세한 내용은 페이지네이션 컴포넌트 참고 (페이지네이션 컴포넌트 바로가기)

  • • 데이터 입력형은 링크 요소를 포함하지 않도록 한다.

  • 한 테이블안에 입력형 요소와 링크가 함꼐 있으면 사용자는 각 요소가 어떤 기능을 하는지 명확하게 파악하기 어려울 수 있으므로 같이 사용하지 않도록 한다.
